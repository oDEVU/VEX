PhysicsComponent ​

The PhysicsComponent defines the physical properties of an entity, such as its shape, mass, friction, and bounciness. It acts as the configuration data that the PhysicsSystem uses to build the actual Jolt Physics body.

Constructors (Factory Methods) ​

The component provides static factory methods to easily create different physics shapes.

Basic Shapes ​

cpp
// Box: Half-extents (0.5 means a 1x1x1 box), BodyType, Mass, Friction, Bounce
auto box = vex::PhysicsComponent::Box({0.5f, 0.5f, 0.5f}, vex::BodyType::DYNAMIC);

// Sphere: Radius
auto sphere = vex::PhysicsComponent::Sphere(1.0f, vex::BodyType::DYNAMIC);

// Capsule: Radius, Height
auto capsule = vex::PhysicsComponent::Capsule(0.5f, 1.0f, vex::BodyType::DYNAMIC);

// Cylinder: Radius, Height
auto cylinder = vex::PhysicsComponent::Cylinder(0.5f, 1.0f, vex::BodyType::STATIC);

Complex Shapes ​

For complex geometry, you can generate colliders from meshes or point clouds.

cpp
// Mesh Collider: Generates a static mesh collider from an existing MeshComponent
// Note: Restricted to STATIC bodies, if you use Dynamic, engine will try to use it as convex hull which may fail if mesh has loose geometry.
auto meshCollider = vex::PhysicsComponent::Mesh(myMeshComponent, vex::BodyType::STATIC);

// Convex Hull: Wraps a set of points (like a shrink wrap)
auto hull = vex::PhysicsComponent::ConvexHull(pointList, vex::BodyType::DYNAMIC);

Collision Callbacks ​

You can bind lambda functions to react to collision events directly within the component.

cpp
// Triggered once when contact begins
physicsComp.addCollisionEnterBinding([this](entt::entity self, entt::entity other, const vex::CollisionHit& hit) {
    log("Bonk! Hit object at position: %f, %f, %f", hit.position.x, hit.position.y, hit.position.z);
});

// Triggered every physics tick while touching
physicsComp.addCollisionStayBinding([](entt::entity self, entt::entity other, const vex::CollisionHit& hit) {
    // e.g., Apply damage while standing in fire
});

// Triggered once when contact ends
physicsComp.addCollisionExitBinding([](entt::entity self, entt::entity other) {
    log("Contact lost");
});

Runtime Updates & Life Span ​

WARNING

Configuration vs Simulation:

The properties inside PhysicsComponent (like mass, friction, shape) are primarily used during body creation.

Changing physicsComponent.mass = 50.0f after the game has started will not automatically update the physics engine. The PhysicsSystem has already built the internal Jolt body based on the initial values.

To update physics properties at runtime, you must use the Physics System:

cpp
// WRONG: This will not update the active physics simulation
comp.mass = 10.0f; 

// CORRECT: Use the Physics System to update the active body
physicsSystem.SetGravityFactor(comp.bodyId, 2.0f);
physicsSystem.SetFriction(comp.bodyId, 0.8f);
physicsSystem.AddForce(comp.bodyId, glm::vec3(0, 100, 0));

If you must change the Shape or Body Type at runtime, you will need to force the system to recreate the body:

cpp
// Update the component data
comp.shape = vex::ShapeType::SPHERE;
// Force the system to rebuild it
physicsSystem.RecreateBodyForEntity(entity, comp);

Accessing the Body ID ​

Once the PhysicsSystem processes the component, it assigns a BodyID. You will need this ID for almost all physics operations (adding force, setting velocity, etc.).

cpp
if (!physicsComp.bodyId.IsInvalid()) {
    auto id = physicsComp.bodyId;
    // Use id with PhysicsSystem...
}

INFO

You can simply imagine Jolt as a oversized, overkill physics system for engines physics components.
