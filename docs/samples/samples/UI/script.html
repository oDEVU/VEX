Interaction & Scripting ​

Once a layout is loaded, you can interact with specific elements by referencing their id defined in the JSON file.

Modifying Widget Properties ​

Beyond simple text updates, you can modify almost any property of a widget at runtime. This allows for dynamic visual feedback, such as changing colors on low health or swapping icons.

cpp
// Change the color of a label (e.g., turn red)
ui->setColor("health_bar", glm::vec4(1.0f, 0.0f, 0.0f, 1.0f));

// Resize a widget dynamically
ui->setSize("health_bar", 200.0f, 20.0f);

// Change an image source (texture auto-loads)
ui->setImage("weapon_icon", "Assets/icons/shotgun.png");

// Move a widget (sets Left/Top yoga properties)
ui->setPosition("floating_text", 500.0f, 100.0f);

// Change font properties
ui->setFont("score_label", "Assets/ui/fonts/pixel.ttf", 24.0f);

// Update borders
ui->setBorder("pause_menu", 2.0f, glm::vec4(1.0f, 1.0f, 1.0f, 1.0f));

Animations ​

Because setters are efficient and hit-detection supports transformations, you can animate widgets inside your GameObject's Update loop.

cpp
void Player::Update(float deltaTime) {
    auto ui = GetComponent<UiComponent>().m_vexUI;
    m_totalTime += deltaTime;

    // 1. Rotation Animation (Spin a logo)
    // The hit-box rotates with the widget, so buttons remain clickable while spinning.
    float rotation = std::sin(m_totalTime * 2.0f) * 15.0f;
    ui->setRotation("logo", rotation);

    // 2. Scale Animation (Pulse effect)
    float scale = 1.0f + (std::sin(m_totalTime * 4.0f) * 0.1f);
    ui->setSize("logo", 96.0f * scale, 64.0f * scale);
    
    // 3. Color Tweening
    if (health < 30) {
        ui->setColor("health_text", glm::vec4(1, 0, 0, 1)); // Red alert
    }
}

Handling Button Clicks ​

Use setOnClick to bind C++ lambda functions to UI buttons.

cpp
// Bind functionality to the "pause" button defined in json
ui->setOnClick("pause", [this]() {
    this->pauseGame();
});

// Bind functionality to a "resume" button in a different menu
ui->setOnClick("resume", [this]() {
    this->resumeGame();
});

Updating Text Dynamically ​

You can update labels in real-time, such as creating an FPS counter or updating player stats.

cpp
void Player::Update(float deltaTime) {
    // Calculate FPS
    std::string fpsString = "FPS: " + std::to_string((int)(1.f/deltaTime));

    // Update the label with id "fps"
    GetComponent<UiComponent>().m_vexUI->setText("fps", fpsString);
}

Toggling Visibility ​

You can show or hide the entire UI canvas without unloading it by modifying the component's visible property.

cpp
// Example: Toggle UI with the 'F' key
inputComp.addBinding(SDL_SCANCODE_F, vex::InputActionState::Pressed, [this](float) {
    auto& uiComp = GetComponent<UiComponent>();
    uiComp.visible = !uiComp.visible;
});

WARNING

When switching between Game and UI interactions, ensure you update the Input Mode. Use m_engine.setInputMode(vex::InputMode::UI) to unlock the cursor for buttons, and switch back to vex::InputMode::Game to return control to the camera/player.
